12 /*
13  * This driver allows applications to store data in the information memory flash
14  * without interfering with each other (except for the total available memory) or
15  * having to care about the characteristics of flash memory.
16  *
17  * infomem_ready() has to be called before any other function does work (except
18  * infomem_init, but use infomem_ready to check if infomem_init is really needed).
19  *
20  * Single applications should only use the infomem_app_*() functions ary infomem_space(),
21  * the rest of the functions should be used only in the global part of the firmware
22  * (or in a dedicated application which's function it is to do memory maintenance tasks).
23  *
24  * Use infomem_app_replace() with count>0 to initialize memory for application. This has
25  * to be redone if data of size zero was present and the application header was therefore
26  * also removed (replace with zero count, clear, delete with zero offset or modify with
27  * zero count and offset).
28  *
29  * All pointers and addresses have to be word addresses (even numbers) and all counts
30  * are given in units of words (two bytes).
31  */
